# 《切线冲刺》视觉升级设计文档

日期：2026-05-02

## 目标

本阶段目标是把《切线冲刺》从方块原型升级为可读、统一、有速度感的移动端 2D 跑酷游戏画面。视觉方向采用 **霓虹日落街机**：暖色日落背景、冷色霓虹赛道、高对比危险物、轻量玻璃感 HUD。

本阶段优先使用程序化图形、Phaser tween、粒子和 CSS 动效完成视觉升级，不引入正式图片素材，不生成 sprite strip，不改变核心玩法规则。

## 视觉方向

整体气质是 **霓虹都市跑酷 + 日落街机赛道** 的混合：

- 保留霓虹都市的速度感、发光反馈、强烈危险信号。
- 引入日落街机的暖色背景、清爽色块和移动端可读性。
- 避免纯黑赛博朋克导致画面压暗。
- 避免过度卡通导致危险感和速度感不足。

关键词：

- 日落
- 霓虹
- 速度线
- 街机
- 高对比
- 清晰反馈
- 低遮挡 HUD

## 色彩系统

建议使用 CSS 变量和 Phaser 主题常量共享同一套色彩意图。

核心色彩：

- 背景上层：橙紫日落 `#ff9f5a`、`#b45cff`
- 背景下层：深蓝夜色 `#101820`、`#172638`
- 跑道底色：深蓝灰 `#1c2b35`
- 轨道霓虹线：青色 `#18f7ff`、紫色 `#8b5cf6`
- 玩家核心：亮黄 `#f6ff5d` + 青色 `#18f7ff`
- 能量球：青黄 `#63d2ff`、`#f6ff5d`
- 护盾：绿色 `#7ee787`
- 磁吸冲刺：紫蓝 `#b78cff`
- 普通障碍：橙色 `#f4a261`
- 危险障碍：品红红 `#ef476f`
- HUD 文本：近白 `#f8fafc`

色彩规则：

- 奖励物使用冷亮色。
- 危险物使用暖红和品红。
- 跑道使用深色承托发光。
- HUD 不使用大面积高饱和背景，避免盖住赛道。

## 画面层级

从后到前：

1. 日落背景渐变
2. 远景城市剪影或抽象低多边形轮廓
3. 三轨跑道底板
4. 纵向速度网格和霓虹轨道线
5. 障碍、道具、能量球
6. 玩家与玩家拖尾
7. 粒子、拾取线、护盾环、撞击反馈
8. DOM HUD 和菜单
9. PC 调试面板

正常游玩时，HUD 只能占据顶部和边缘区域，不能覆盖角色附近和下半部核心判断区。

## 游戏内元素

### 玩家

当前矩形替换为程序化“能量滑行者”：

- 圆角主体。
- 中央发光核心。
- 底部短尾焰。
- 换轨时向目标方向轻微倾斜。
- 疾跑时拖尾拉长。

第一阶段仍使用 Phaser Graphics 或 Rectangle 组合，不使用图片 sprite。

### 跑道

当前大矩形跑道替换为：

- 深蓝底板。
- 三条明确轨道。
- 青色和紫色霓虹边线。
- 纵向速度网格线。
- 疾跑时轨道线亮度提升。

PC 宽屏下仍保持手机比例视窗，不拉宽轨道。

### 道具

能量球：

- 发光圆球。
- 轻微脉冲缩放。
- 拾取时向玩家吸入或闪烁消失。

护盾：

- 绿色六边形或环形符号。
- 拾取后玩家周围出现短暂护盾环。

磁吸冲刺：

- 紫蓝能量晶体。
- 触发后出现吸附线或短暂能量波。

### 障碍

普通路障：

- 橙色宽厚块。
- 有警示边和轻微发光。

矮栏：

- 更扁、更低。
- 红橙色。
- 视觉暗示需要跳跃。

横梁：

- 横向紫色霓虹梁。
- 视觉暗示需要滑铲。

危险区：

- 品红裂隙或断路。
- 危险等级最高。
- 不能和普通障碍混淆。

形状必须帮助区分障碍类型，不能只靠颜色。

## UI 与 HUD

UI 使用 DOM + CSS，不切成图片。

### HUD

当前顶部信息保留，但升级为轻量玻璃霓虹风格：

- 分数、距离、护盾使用半透明胶囊。
- 疾跑槽使用细长能量条。
- 疾跑槽满或疾跑中发光增强。
- 暂停按钮保留右上角，尺寸适合触摸。
- 文本保持短、清晰、大号。

### 开始页

开始页应包含：

- 游戏名“切线冲刺”。
- 最高分。
- 开始按钮。
- 背后仍能看到霓虹日落赛道或抽象背景。

开始页不做营销式大段说明。

### 暂停页

暂停页使用深色半透明覆盖层：

- 继续。
- 重开。
- 回到开始页可留到后续阶段。

暂停层不需要复杂装饰。

### 结算页

结算页展示：

- 本局分数。
- 距离。
- 最高连击。
- 最高分。
- 再来一局。

刷新最高分时可加入小型发光提示，但不阻塞重开。

### PC 调试面板

PC 调试面板继续保留，但视觉上弱化：

- 深色半透明背景。
- 小字号。
- 不抢主游戏视窗的视觉焦点。
- `?debug=1` 和 F2 行为保持不变。

## 动画反馈

动画原则：

- 用于说明状态变化。
- 不做持续干扰。
- 强动效只用于奖励、危险、疾跑和失败。
- 尊重 `prefers-reduced-motion`，非必要 CSS 动效可降低或关闭。

基础动效：

- 换轨：玩家倾斜、短拖尾、目标轨道闪烁。
- 跳跃：玩家压缩后弹起，影子变小，落地冲击波。
- 滑铲：玩家高度压低，拖尾贴地。
- 拾取能量：能量球闪烁或吸入，HUD 分数轻跳。
- 护盾拾取：玩家出现绿色护盾环。
- 撞击：轻微屏幕震动、玩家闪白、护盾破碎环。
- 疾跑：赛道线变亮、边缘速度线出现、玩家拖尾拉长。
- 失败：红色冲击，结算面板淡入。

## 实现范围

包含：

- 霓虹日落背景和三轨赛道重绘。
- 玩家、障碍、道具的程序化形状替换。
- HUD、开始页、暂停页、结算页视觉升级。
- 换轨、跳跃、滑铲、拾取、受伤、疾跑、失败的基础动画反馈。
- PC 调试面板保留并弱化视觉权重。
- 预留后续 sprite 和图片素材接入点。

不包含：

- AI 生成正式角色图。
- sprite strip 动画。
- 背景大图。
- 商店、皮肤、任务系统。
- 新玩法、新道具、新障碍规则。
- 在线排行榜或后端。

## 模块边界

新增模块：

- `src/game/visual/theme.ts`
  - 管理霓虹日落主题颜色、尺寸、发光强度、动画时长。

- `src/game/visual/GameVisualFactory.ts`
  - 创建跑道、玩家、障碍、道具的 Phaser 视觉对象。
  - 当前使用程序化图形。
  - 后续可在内部替换为 sprite。

- `src/game/visual/EffectController.ts`
  - 管理拾取闪光、撞击震动、疾跑速度线、护盾环、换轨闪烁等短反馈。

修改模块：

- `src/game/scenes/GameScene.ts`
  - 继续负责场景生命周期和调用视觉工厂。
  - 不再散落大量颜色、尺寸和形状常量。

- `src/styles.css`
  - 增加 CSS 变量、HUD 胶囊、按钮、菜单面板、动效和移动端适配。

保持玩法规则不变：

- `src/game/state.ts`
- `src/game/collision/CollisionSystem.ts`
- `src/game/spawn/ObstacleSpawner.ts`
- `src/game/lane/LaneController.ts`
- `src/game/input/InputController.ts`

除非视觉反馈必须暴露某个状态，否则不修改玩法规则模块。

## 资产策略

第一轮不生成正式图片素材。理由：

- 当前最缺的是动效、层级和可读性。
- 程序化视觉更容易快速调整尺寸和颜色。
- 移动端可读性需要先在真实游戏视窗里验证。
- 过早生成素材会让碰撞尺寸、动画节奏和视觉风格被切图限制。

后续素材生成顺序：

1. 玩家角色 seed frame。
2. 玩家跑步、跳跃、滑铲 sprite strip。
3. 障碍和道具 icon 或 cutout。
4. 背景远景层。
5. 菜单主视觉或宣传图。

后续若使用 AI 生成：

- 使用 `imagegen` 生成 seed frame 或道具 cutout。
- 透明图优先走纯色背景 + 本地抠图。
- 角色动画使用 `sprite-pipeline`，从已批准 seed frame 开始生成整条动画 strip。

## 验收标准

功能不回退：

- 游戏仍能开始、失败、结算、再来一局。
- 鼠标手势、触摸手势、键盘输入都保留。
- PC 调试面板仍可用。
- 本地最高分仍正常保存。

视觉可读：

- 玩家位置在 3 条轨道上清楚可见。
- 能量球、护盾、磁吸、普通障碍、矮栏、横梁、危险区能一眼区分。
- HUD 不遮挡跑道核心判断区。
- 疾跑状态明显但不刺眼。
- 受伤和失败反馈清楚。

移动端适配：

- 9:16 视窗保持主要视觉。
- 小屏下按钮文字不溢出。
- 安全区顶部 HUD 不挤压。
- 动效不会造成持续遮挡。

工程验证：

- `npm test` 通过。
- `npm run build` 通过。
- `http://127.0.0.1:5173/` 可打开。
- `http://127.0.0.1:5173/?debug=1` 可打开。
- 浏览器截图检查普通页和 debug 页。
- 提交日志继续使用中文。
