# 《切线冲刺》游戏设计文档

日期：2026-05-02

## 目标

《切线冲刺》是一款移动端优先的竖屏 2D 短局跑酷游戏。玩家在 3 条轨道之间快速切换，通过左右滑动、上滑跳跃、下滑滑铲来躲避障碍、收集能量、维持连击，并在 3-5 分钟内完成一局高节奏挑战。

游戏优先服务移动端体验，同时必须支持 PC 端方便调试和预览。PC 端需要同时提供键盘输入和鼠标手势模拟，让开发者可以在桌面浏览器里快速验证移动端手感。

## 玩家幻想

玩家扮演高速奔跑的角色，在不断变窄的安全路径中做出瞬间反应。游戏的快感来自清晰、短促、连续的判断：

- 看到障碍组合。
- 立即选择轨道或动作。
- 成功擦身而过。
- 收集能量并延续连击。
- 节奏越来越快，风险越来越高。

游戏不追求复杂叙事。第一版重点是“看得懂、反应快、重开爽”。

## 核心循环

单局循环：

1. 玩家点击开始，角色自动向前奔跑。
2. 障碍和奖励沿 3 条轨道生成并向玩家接近。
3. 玩家通过滑动手势执行换轨、跳跃、滑铲。
4. 成功躲避障碍并收集能量球，获得分数和连击。
5. 连续无伤会提升连击，并积累疾跑槽。
6. 受伤会消耗护盾并清空连击；护盾耗尽后再次受伤则结束本局。
7. 本局结束后进入结算页，展示分数、距离、最高连击和是否刷新最高分。
8. 玩家一键重开。

长期循环：

1. 完成一局。
2. 获得金币或经验。
3. 解锁角色皮肤、轨道主题或轻量强化。
4. 再次挑战更高分。

MVP 阶段只实现单局循环、本地最高分和一键重开。金币、皮肤、主题、成长升级留到第二阶段。

## 单局节奏

每局目标时长为 3-5 分钟。第一版可以先做无限跑道，难度随时间递增，不强制终点。

节奏分 3 段：

1. 热身段：0-30 秒
   - 障碍稀疏。
   - 主要训练左右换轨、跳跃和滑铲。
   - 能量球排列比较友好。

2. 加压段：30 秒到 2 分钟
   - 出现组合障碍。
   - 玩家需要连续执行两个或多个动作。
   - 能量球开始引导高风险路线。

3. 冲刺段：2 分钟后
   - 速度持续提升。
   - 障碍密度增加。
   - 连击收益更高。
   - 错误窗口更小，但路线仍必须可解。

## 轨道与动作

游戏采用固定 3 轨道：

- 左轨
- 中轨
- 右轨

角色默认位于中轨。左右换轨是离散动作，不使用自由横向移动。

基础动作：

- 左滑：向左换一条轨道。
- 右滑：向右换一条轨道。
- 上滑：跳跃。
- 下滑：滑铲。
- 短点击：确认、开始、继续。

动作约束：

- 一次手势只触发一个动作。
- MVP 允许跳跃中换轨，但滑铲期间不允许换轨，以降低状态复杂度。
- 换轨动作应短促清晰，避免拖泥带水。

## 输入设计

输入统一通过 Pointer Events 和键盘事件转换为游戏动作。玩法逻辑只接收动作，不直接关心输入设备。

动作枚举：

- laneLeft
- laneRight
- jump
- slide
- pause
- restart
- confirm

移动端输入：

- 触摸左滑：laneLeft
- 触摸右滑：laneRight
- 触摸上滑：jump
- 触摸下滑：slide
- 短点击：confirm

PC 鼠标手势模拟：

- 鼠标左键按下并快速左划：laneLeft
- 鼠标左键按下并快速右划：laneRight
- 鼠标左键按下并快速上划：jump
- 鼠标左键按下并快速下划：slide
- 短点击：confirm

PC 键盘调试输入：

- A 或左方向键：laneLeft
- D 或右方向键：laneRight
- W 或上方向键：jump
- S 或下方向键：slide
- Esc：pause
- R：restart
- Enter 或空格：confirm

手势判定规则：

- 设置最小滑动距离，避免短点击误判为手势。
- 设置最大手势时间，避免慢拖误判为快速滑动。
- 横向位移明显大于纵向位移时判定为左右滑。
- 纵向位移明显大于横向位移时判定为上下滑。
- 手势触发后立即锁定方向，直到本次指针释放。

## 障碍设计

MVP 障碍分 4 类：

1. 路障
   - 占据单条轨道。
   - 需要左右换轨躲避。
   - 是最基础的节奏障碍。

2. 矮栏
   - 占据单条或双条轨道。
   - 需要跳跃越过。
   - 常用于制造“先换轨再跳”的组合。

3. 横梁
   - 横跨一条到三条轨道。
   - 需要滑铲通过。
   - 用来打破玩家只依赖跳跃的惯性。

4. 断路或危险区
   - 占据单条轨道。
   - 触碰后直接失败或造成重罚。
   - 出现频率低，用于制造紧张感。

生成原则：

- 同一时间窗口内必须至少存在一条可通过路线。
- 热身段不生成需要连续 3 个动作才能通过的组合。
- 冲刺段可以生成短窗口组合，但不能依赖像素级反应。
- 能量球可以引导玩家走高分路线，但不应该强迫玩家走危险路线。

## 道具设计

MVP 道具分 3 类：

1. 能量球
   - 基础收集物。
   - 提供分数。
   - 维持或提升连击。

2. 护盾
   - 恢复 1 层护盾。
   - 出现频率低。
   - 不能让玩家长期无脑容错。

3. 磁吸冲刺
   - 短时间吸附附近能量球。
   - 略微提升速度。
   - 是奖励，同时把玩家推入更高节奏。

疾跑槽和磁吸冲刺使用同一个基础效果：

- 连续收集能量球会积累疾跑槽。
- 疾跑槽满后自动触发 3 秒磁吸冲刺。
- 磁吸冲刺道具可以直接触发同一效果。
- MVP 只需要实现基础吸附和轻微加速，复杂特效留到第二阶段。

## 得分与失败

分数来源：

- 距离分。
- 能量球分。
- 连击倍率。

连击规则：

- 连续收集能量球并保持无伤会提升连击。
- 受伤会清空连击。
- 连击倍率影响能量球得分。

护盾规则：

- 玩家默认有 1 层护盾。
- 撞到普通障碍时，如果有护盾，扣除 1 层护盾并清空连击。
- 护盾为 0 时再次撞到普通障碍，本局结束。
- 断路或危险区可以直接结束本局。

## UI 与 HUD

HUD 保持低干扰，不遮挡跑道判断。

局内 HUD：

- 顶部左侧：当前分数。
- 顶部中间：距离或进度。
- 顶部右侧：护盾状态。
- 角色附近或顶部细条：连击倍率和疾跑槽。
- 右上角：暂停按钮。

菜单页面：

- 开始页：开始游戏、最高分、设置入口。
- 暂停页：继续、重开、回到开始页。
- 结算页：本局分数、距离、最高连击、最高分状态、一键重开。

UI 实现建议：

- 游戏跑道、角色、障碍、道具使用 Phaser Canvas/WebGL 渲染。
- HUD、菜单、结算页使用 DOM 层。
- DOM 层读取可序列化状态，不直接改 Phaser 渲染对象。

## 移动端适配

目标画幅为 9:16 竖屏。

适配原则：

- 跑道保持固定比例，横向居中。
- PC 宽屏预览时显示手机比例视窗，两侧留安全边。
- 不因为宽屏而拉宽 3 条轨道。
- HUD 避开系统安全区和跑道核心判断区。
- 触摸目标足够大，暂停按钮不能过小。

MVP 不强制横屏支持。横屏时可以显示居中的竖屏游戏视窗。

## PC 调试与预览

PC 端用于开发、调试和桌面预览，不改变核心移动端玩法。

PC 端必须支持：

- 键盘输入。
- 鼠标手势模拟。
- 手机比例游戏视窗。
- 一键重开。

开发调试面板在 `?debug=1` 或快捷键 F2 时显示。

调试面板内容：

- 当前速度。
- 障碍生成频率。
- 当前距离。
- 当前分数。
- 当前连击倍率。
- 当前输入事件。
- 护盾状态。
- 疾跑槽。
- 碰撞框显示开关。
- 慢速模式开关。
- 随机种子显示和重置。

正式移动端运行时隐藏调试面板。

## 技术方案

推荐技术栈：

- Phaser 3
- TypeScript
- Vite
- HTML/CSS DOM Overlay

Phaser 负责游戏本体：

- 场景生命周期。
- 角色和障碍渲染。
- 动画。
- 碰撞检测。
- 资源加载。
- 游戏循环。

Vite 和 TypeScript 负责工程体验：

- 本地开发服务器。
- 热更新。
- 类型检查。
- 打包构建。

DOM Overlay 负责：

- 开始页。
- HUD。
- 暂停页。
- 结算页。
- 调试面板。

## 模块划分

建议模块：

- `GameScene`
  - 管理 Phaser 场景生命周期、对象池、碰撞检测、动画播放和场景更新。

- `RunState`
  - 保存可序列化的局内状态，包括分数、距离、速度、护盾、连击、疾跑槽、暂停和结束状态。

- `InputController`
  - 统一键盘、触摸和鼠标手势，把物理输入转换为动作。

- `LaneController`
  - 管理 3 条轨道位置、角色换轨、跳跃、滑铲和状态约束。

- `ObstacleSpawner`
  - 根据时间、速度和随机种子生成障碍、能量球和道具组合。

- `CollisionSystem`
  - 判断玩家与障碍、道具、危险区的交互结果，并更新 `RunState`。

- `HudOverlay`
  - DOM 层 HUD，读取状态并刷新显示。

- `DebugPanel`
  - 仅开发环境启用，显示调试数据并提供调试开关。

## 数据流

整体数据流：

1. 输入层捕获触摸、鼠标或键盘事件。
2. `InputController` 输出标准动作。
3. `LaneController` 根据动作更新玩家状态。
4. `GameScene` 推进场景、移动对象、播放动画。
5. `CollisionSystem` 检查碰撞并更新 `RunState`。
6. `HudOverlay` 读取 `RunState` 刷新界面。
7. 本局结束后进入结算页。

原则：

- 渲染对象不是真相来源。
- 可保存状态放在 `RunState`。
- 设置、最高分和解锁状态使用 `localStorage`。
- 调试面板只能通过公开调试接口改动状态，不直接操作渲染对象内部字段。

## MVP 范围

第一阶段必须包含：

- Phaser 3 + TypeScript + Vite 项目骨架。
- 竖屏 3 轨跑道。
- 玩家自动前进。
- 左右换轨、跳跃、滑铲。
- 触摸手势输入。
- 鼠标手势模拟。
- 键盘调试输入。
- 4 类基础障碍。
- 能量球。
- 护盾。
- 疾跑槽和基础磁吸冲刺效果。
- 分数、距离、连击。
- 本地最高分。
- 开始页、HUD、暂停页、结算页。
- PC 调试面板。

第一阶段不包含：

- 账号系统。
- 在线排行榜。
- 付费系统。
- 多角色技能。
- 复杂皮肤商店。
- 多关卡地图。
- 后端服务。

## 测试与验收

功能验收：

- 手机上可以通过滑动完成全部核心动作。
- PC 上可以通过鼠标手势模拟手机操作。
- PC 上可以通过键盘完成调试操作。
- 一局可以完整开始、游玩、失败、结算、重开。
- 最高分可以保存在本地。

手感验收：

- 短点击不会频繁误判为滑动。
- 快速滑动不会漏判。
- 换轨反馈清晰。
- 跳跃和滑铲的判定窗口可理解。
- 失败原因清楚，不显得随机。

关卡验收：

- 热身段不会过难。
- 2 分钟后难度明显上升。
- 障碍组合不生成无解路线。
- 能量球路线能自然引导玩家冒险。

界面验收：

- 移动端 HUD 不遮挡跑道判断。
- PC 预览保持手机比例。
- 调试面板只在开发模式或 debug 参数下出现。
- 结算页一键重开路径明显。

## 后续扩展

第二阶段可以加入：

- 金币和经验。
- 角色皮肤。
- 跑道主题。
- 每日挑战。
- 简单任务系统。
- 更多障碍组合。
- 磁吸冲刺完整特效。
- 种子复现和回放调试。

这些扩展不应阻塞 MVP。第一阶段的核心目标是验证短局跑酷手感、输入一致性和移动端适配。
